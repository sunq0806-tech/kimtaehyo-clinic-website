1. Executive Summary

프리타임은 “강사가 예약 가능한 시간을 만들고 링크로 공유하면 학생이 회원가입 없이 예약한다”는 단순한 가치 제안을 가진 예약 관리 SaaS 랜딩 페이지다.

공개 페이지 기준으로 서비스 메시지는 명확하다. 핵심 온보딩은 무료 가입 → 일정 설정 → 링크 공유의 3단계로 요약되며, 주요 타깃은 과외·음악·PT·미술·어학원·IT 강사처럼 1:1 또는 소규모 수업 예약을 반복적으로 처리하는 개인 강사다. 가격은 무료/프로 2단 구조로 이해하기 쉽고, 베타 기간에는 프로 기능 무료 이용 메시지를 전면에 둔다.

핵심 강점

  • 메인 메시지가 짧고 명확하다.
  • 강사 입장의 시작 절차를 3단계로 단순화했다.
  • 학생 회원가입 없음, Google Calendar 연동, 이메일/Discord 알림 등 구매 전 의문을 FAQ에서 다룬다.
  • 약관·개인정보처리방침에서 Google Calendar 읽기 전용 접근 범위를 설명해 신뢰를 보강한다.

핵심 리스크

  • 가격 카드와 FAQ의 무료 플랜 제한 수량이 다르다.
  • 실제 캘린더/예약 페이지 UI 증거가 부족하다.
  • CTA 클릭 후 단계, 로그인/회원가입/예약 폼 화면이 공개 페이지에서 확인되지 않는다.
  • 지원 메뉴의 실제 하위 화면이 확인되지 않아 구매 전 문의 동선이 약하다.

2. 분석 방법 및 확인 범위

수집 방식공개 URL 접근, 메인 페이지 텍스트 구조 추출, 푸터 법적 고지 페이지 이동 확인, 약관/개인정보처리방침 주요 내용 확인
확인된 페이지메인 페이지, 이용약관, 개인정보처리방침, 로고 이미지
미확보 화면실제 브라우저 전체 PNG 캡처, 로그인/회원가입 후 화면, 강사용 대시보드, 학생용 예약 폼, CTA 클릭 후 상태, 입력 폼 검증 상태
보고서 내 화면 자료 성격아래의 화면 보드는 실제 브라우저 스크린샷이 아니라, 접근 가능한 공개 텍스트를 기반으로 재구성한 화면 구조 맵이다. 실제 캡처가 필요한 항목은 Appendix의 누락 가능 화면 목록에 별도로 정리했다.

PC와 모바일 차이 분석

공개 텍스트 기준으로 PC와 모바일의 IA 및 콘텐츠 차이는 확인되지 않는다. 모바일 최적화는 FAQ에서 명시되어 있으나 실제 모바일 렌더링 캡처가 확보되지 않았으므로 레이아웃 차이는 추후 확인 항목이다.

  • PC에서는 카드형 그리드가 정보 탐색에 유리할 가능성이 높다.
  • 모바일에서는 카드가 세로 스택으로 길어져 CTA 회수가 중요하다.
  • 모바일에서는 하단 고정 CTA, FAQ 접힘/펼침, 가격 카드 비교 가독성이 전환에 직접 영향을 줄 수 있다.
  • 모바일 네비게이션이 햄버거/드로어인지, 헤더 CTA가 유지되는지는 실제 캡처로 확인해야 한다.

그림 2. 모바일 구조 맵. 실제 모바일 캡처는 미확보이며, 공개 콘텐츠를 모바일 스택 구조로 재배치한 분석용 맵.

8. 입력폼 및 오류 상태 분석

공개 페이지에서는 로그인/회원가입/문의/예약 입력폼의 실제 UI가 확인되지 않았다. 따라서 입력 전·입력 중·오류 상태는 실화면 근거가 아니라 추후 반드시 수집해야 할 검증 항목으로 정리한다.

폼 후보입력 전 확인 항목입력 중 확인 항목오류 상태 확인 항목
회원가입/로그인이메일, 소셜 로그인, 약관 동의, Google 권한 범위필드 포커스, 입력값 유지, 비밀번호 보기/숨김빈 값, 이메일 형식 오류, 중복 계정, OAuth 취소

Appendix B. 최종 판단

서비스 기획자 관점의 결론: 프리타임은 MVP 단계 랜딩으로는 메시지와 타깃이 선명하다. 그러나 경쟁사 UX/UI 분석 관점에서는 제품 화면 증거, 가입 이후 플로우, 학생 예약 경험, 입력 검증 상태가 아직 보이지 않아 “실제로 얼마나 쉬운지”를 검증하기 어렵다.

가장 먼저 해결해야 할 것은 무료 플랜 정보 불일치와 CTA 후속 단계의 불명확성이다. 그다음 실제 예약 페이지 데모와 캘린더 화면을 보여주면, “간편한 예약 관리”라는 핵심 약속의 신뢰도가 크게 올라갈 것이다.
