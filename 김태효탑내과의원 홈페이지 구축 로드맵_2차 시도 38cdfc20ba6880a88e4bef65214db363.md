# 김태효탑내과의원 홈페이지 구축 로드맵

## 프리타임 구조를 참고한 병원 소개·예약 서비스 기획안

## 0. 한 줄 결론

**김태효탑내과의원은 기존 네이버 블로그에 흩어져 있는 병원소개, 건강검진, 내시경, 초음파, 웰체크 콘텐츠를 기반으로, 프리타임처럼 “명확한 가치 제안 → 3단계 예약 흐름 → 주요 기능/진료 안내 → FAQ → CTA” 구조의 병원 예약형 홈페이지로 확장하는 것이 적합합니다.**

---

## 1. 현재 김태효탑내과 모바일 블로그 진단

모바일 캡처 기준 첫 화면에는 병원명, 프로필, 이웃추가 버튼, 소개, 인기글이 보입니다. 소개에는 문의전화 `055-812-8275`, 사업자번호, 주소가 텍스트로 들어가 있고, 인기글에는 초음파 검사, 병원 인사말, 내시경 검사, 오시는길이 노출됩니다.

현재 블로그의 강점은 콘텐츠가 이미 있다는 점입니다. 초음파, 내시경, 건강검진, 웰체크, 오시는길, 병원소개가 존재합니다. 다만 모바일에서는 사용자가 바로 해야 할 행동, 즉 **전화하기, 예약하기, 진료항목 확인하기, 위치 보기**가 강한 버튼 구조로 설계되어 있지 않습니다.

| 현재 상태 | 문제 |
| --- | --- |
| 소개 영역에 전화번호와 주소가 있음 | 버튼형 CTA가 아니라서 행동 유도가 약함 |
| 인기글에 주요 콘텐츠가 노출됨 | 사용자가 원하는 진료별 탐색 구조는 약함 |
| 전체글이 오래된 게시글 중심 | 병원이 현재도 활발히 운영 중인지 신뢰 약화 가능 |
| 병원소개·건강검진·내시경·웰체크 콘텐츠 보유 | 홈페이지 핵심 페이지로 재활용 가능 |
| 모바일 블로그 기본 UI 사용 | 병원 전용 예약/문의 경험 설계가 어려움 |

## 2. 프리타임에서 참고해야 할 핵심 구조

업로드된 프리타임 보고서에 따르면 프리타임은 **“강사가 예약 가능한 시간을 만들고 링크로 공유하면 학생이 회원가입 없이 예약한다”**는 단순한 가치 제안을 가진 예약 관리 SaaS입니다. 핵심 온보딩은 **무료 가입 → 일정 설정 → 링크 공유**의 3단계로 요약됩니다.

이 구조를 병원 홈페이지에 적용하면 다음처럼 변환할 수 있습니다.

| 프리타임 구조 | 병원 홈페이지 적용 |
| --- | --- |
| 무료 가입 | 진료/검사 항목 선택 |
| 일정 설정 | 희망 방문일·시간 입력 |
| 링크 공유 | 예약 요청 완료 및 병원 확인 |
| 학생 회원가입 없이 예약 | 환자가 복잡한 가입 없이 예약 요청 |
| 캘린더·예약 관리 | 병원 관리자 예약 요청 확인 |
| FAQ | 검사 전 주의사항, 진료시간, 주차, 예약 변경 안내 |
| 반복 CTA | 전화 문의, 예약 요청, 오시는길 CTA 반복 |

프리타임 보고서에서 중요한 점은 랜딩 페이지가 **1-page IA**, **3-step 시작 플로우**, **가격/FAQ/푸터 구조**를 갖고 있다는 것입니다.

김태효탑내과 홈페이지도 처음에는 복잡한 병원 사이트보다 **단일 랜딩형 홈페이지 + 예약 요청 MVP**로 시작하는 것이 좋습니다.

## 5. 메인 페이지 구성안

## Section 1. Hero

프리타임은 첫 화면에서 “예약 관리, 이제 간편하게”라는 메시지로 문제와 해결을 바로 제시합니다.

김태효탑내과도 첫 화면에서 병원의 핵심 가치를 즉시 보여줘야 합니다.

```
진주 평거동 내과·건강검진·내시경
김태효탑내과의원

정교수 출신 대표원장이 진료하는 지역 주치의 내과

[예약 요청하기] [전화 문의] [오시는 길]
```

Hero에 반드시 들어가야 할 정보는 다음 5개입니다.

| 정보 | 이유 |
| --- | --- |
| 병원명 | 브랜드 인지 |

## Section 2. 3단계 예약 요청

프리타임의 핵심 전환 플로우는 **홈 진입 → 무료로 시작하기 → 가입/인증 → 일정 설정 → 링크 공유 → 학생 예약**입니다.

병원에서는 이를 다음처럼 바꾸는 것이 적합합니다.

```
3단계로 간편하게 예약을 요청하세요

01 진료 항목 선택
건강검진, 내시경, 초음파, 만성질환 관리 중 필요한 항목을 선택합니다.

02 희망 날짜·시간 입력
방문 가능한 날짜와 시간을 남겨주세요.

03 병원 확인 후 안내
병원에서 가능 여부를 확인한 뒤 전화 또는 문자로 안내드립니다.
```

중요한 점은 **“예약 확정”이 아니라 “예약 요청”**으로 설계하는 것입니다. 병원은 검사 준비, 의료진 일정, 당일 진료 상황이 있으므로 실시간 확정 예약보다 병원 확인 후 안내가 안전합니다.

## 6. 예약 서비스 MVP 설계

### 6-1. MVP 핵심 원칙

프리타임은 예약 가능한 시간을 설정하고 링크를 공유하는 구조입니다. 하지만 병원은 의료 서비스이므로 프리타임처럼 완전 자동 예약보다는 **예약 요청 접수 → 병원 확인 → 안내**가 적합합니다.

| 구분 | 프리타임 | 김태효탑내과 MVP |
| --- | --- | --- |
| 예약 주체 | 학생이 직접 예약 | 환자가 예약 요청 |
| 제공자 | 강사 | 병원 |
| 일정 확정 | 시스템에서 자동 처리 가능 | 병원 확인 후 확정 |
| 입력 정보 | 이름, 예약 시간 등 | 이름, 연락처, 진료항목, 희망일 |
| 민감정보 | 낮음 | 건강정보 가능성이 있어 최소 수집 필요 |
| 알림 | 이메일/Discord 등 | 전화, 문자, 관리자 알림 |

### 6-2. 예약 요청 폼

초기 MVP에서는 입력 필드를 최소화해야 합니다.

| 필드 | 필수 여부 | 이유 |
| --- | --- | --- |
| 이름 | 필수 | 예약 확인 |
| 연락처 | 필수 | 병원 연락 |

## 10. 포트폴리오용 프로젝트 정의

### 프로젝트명

**김태효탑내과의원 블로그 기반 병원 예약 홈페이지 UX 기획**

### 문제정의

기존 네이버 블로그는 병원소개, 건강검진, 내시경, 초음파, 웰체크 콘텐츠를 보유하고 있지만, 모바일 사용자가 진료 가능 여부를 빠르게 판단하고 예약 요청까지 이어지는 전환 구조가 약하다.

### 해결방향

프리타임의 1-page 랜딩 IA와 3단계 예약 플로우를 참고해, 병원 맥락에 맞는 **진료 선택 → 희망일 입력 → 병원 확인 후 안내** 구조의 예약 요청 MVP를 설계한다.

### 포트폴리오 문장

**“네이버 블로그에 분산되어 있던 김태효탑내과의원의 병원소개·검진·내시경·초음파·만성질환 콘텐츠를 분석하고, 프리타임의 3단계 예약 구조를 병원 맥락에 맞게 변환하여 모바일 중심 예약 요청형 홈페이지 IA와 MVP 플로우를 설계했습니다.”**

### 면접 답변 초안

**“프리타임의 강점은 복잡한 예약 과정을 3단계로 단순화한 점이었습니다. 저는 이 구조를 병원 홈페이지에 그대로 복사하지 않고, 의료 서비스 특성에 맞게 ‘확정 예약’이 아닌 ‘예약 요청 후 병원 확인’ 구조로 바꾸었습니다. 이를 통해 환자는 가입 없이 진료 항목과 희망 시간을 남길 수 있고, 병원은 일정과 검사 준비사항을 확인한 뒤 안전하게 안내할 수 있도록 설계했습니다.”**
